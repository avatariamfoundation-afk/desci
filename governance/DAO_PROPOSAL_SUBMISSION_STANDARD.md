# DAO_PROPOSAL_SUBMISSION_STANDARD.md

## 1. Purpose
This document defines the mandatory standard for submitting proposals to the NeuroGrid DAO. It ensures that all proposals are structured, reviewable, compliant, and suitable for risk-weighted governance decision-making within a regulated, clinical-grade ecosystem.

## 2. Scope
This standard applies to all DAO proposals, including:
- Governance changes
- Registry updates
- Funding requests
- Policy amendments
- Model lifecycle actions
- Emergency actions requiring ratification
- Infrastructure or protocol modifications

Proposals that do not meet this standard are invalid.

## 3. Submission Authority
Proposals may be submitted only by:
- Core Governance Members
- Authorized Committee Chairs
- Safety Council (including emergency actions)
- Delegated proposers explicitly authorized by governance vote

Unauthorized submissions are automatically rejected.

## 4. Proposal Identification
Each proposal must be assigned:
- Unique Proposal ID
- Submission timestamp
- Proposer identity and role
- Proposal type classification

Proposal IDs are immutable and globally unique.

## 5. Proposal Classification
Every proposal must declare its type:
- Type I – Informational
- Type II – Operational
- Type III – High-Risk / Regulated
- Type IV – Emergency Ratification

Misclassification constitutes a governance violation.

## 6. Mandatory Proposal Structure
All proposals must include the following sections:

### 6.1 Executive Summary
- Clear description of the proposal
- Intended outcome
- Decision requested of the DAO

### 6.2 Rationale
- Justification for the proposal
- Supporting evidence or context
- Alignment with NeuroGrid objectives
